### Ever wondered what kind of network calls your applications make behind the scenes?

It was always known that apps like VS Code, JetBrains IDEs, and even distributions like Ubuntu send telemetry(metrics, usage data, and diagnostics) back to their servers.

Browsers do this too.

Every time you visit a website, multiple background requests are made tracking, analytics, and ads.

But how do you actually know **where your data is going?**

And more importantly, how do you monitor it?

On macOS, tools like _Little Snitch_ have existed for a long time.

They let you see:  
• which apps are making network requests  
• where those requests are going  
• and allow you to control them

Recently, Little Snitch had a initial port to [Linux](https://obdev.at/products/littlesnitch-linux/index.html).

It’s not a full-fledged port yet.

Instead, it’s using **eBPF**, which hooks into the Linux kernel to observe network activity.

Because of this, the Linux version is still limited compared to macOS, but the underlying approach is quite interesting.

The architecture is fairly simple:

• `ebpf` (Rust crate) - contains eBPF programs \
• `common` - shared types between kernel and user space \
• `demo-runner` - loads eBPF programs and demonstrates comm. via eBPF maps \
• `webroot` - a lightweight web UI

What stands out is the use of **eBPF for network visibility**.

Not just allowing or blocking traffic, you can: \

• observe connections **at the syscall or socket level** (who is talking to whom, not just packets)\
• track events in real time without copying packets to user space\
• correlate network activity with **process context (PID, binary, container, etc.)**\
• apply filtering and aggregation **directly in kernel space** before data is transferred

It’s still early in development and we can expect more stuff to come in the near future, but it’s a neat tool for anyone interested in privacy, networking or systems-level behavior.

A good reminder:

Most systems are doing more in the background than we realize.

Sometimes it’s worth taking a look. 👀
